## Plan

### Duplicate Slice

**Steps**
1. On export, initialize a table to track exported slice names and their used increments.
2. For each slice to export:
   - Check the table for existing entries for this name.
   - Find the lowest available increment (starting at 0) not already used for this name.
   - Construct the filename:
     - If increment is 0: `name.png`
     - If increment > 0: `name_increment.png`
   - Add the name and increment to the tracking table.
   - Export the slice using the constructed filename.
3. Repeat for all slices, ensuring no filename is reused during this export session.

## Future Improvements

### Duplicate Slice Handling

- **Configurable Initial Increment:**  
  Allow users to choose whether the initial increment for duplicate slices starts at 0 or 1. Revisit after plugin settings implmeneted. This would affect whether the first exported file is named `name.png`/`name_0.png` or `name_1.png`, and would require parameterizing the increment logic in the export and utility functions.

- **Sanitize Filenames:**  
  Implement logic to sanitize slice names and user data used in filenames, removing or replacing characters that are not allowed by the operating system. This will prevent export errors and ensure compatibility across platforms.
